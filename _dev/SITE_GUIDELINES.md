# 技術士化学部門 過去問解説サイト 設計・構成ガイドライン

このドキュメントは、サイトの設計思想、デザインルール、およびディレクトリ構成をまとめたローカル専用のガイドラインです。GitHub等には公開せず、ローカル環境での開発・保守の際の参照用として使用します。

## 1. サイトの目的とコンセプト
- **目的**: 技術士（化学部門）一次試験の過去問題解説を提供し、受験者の学習を支援する。
- **コンセプト**: 学習の妨げにならないよう、シンプルでクリーンなデザイン。装飾（絵文字など）は極力排除し、コンテンツ（問題と解説）に集中できるUIを提供する。

## 2. デザインルール
- **メインカラー (Primary Color)**: `#006938`（ケムファクの公式ロゴカラーである深い緑色）。リンク、プロンプト、ハイライトなどに使用。
- **背景色**: 
  - 全体背景: わずかにグレーがかった白（`#f8fafc`）。
  - 各種カード（問題、解説等）: 真っ白（`#ffffff`）。
- **フォント**: `"Helvetica Neue", Arial, "Hiragino Kaku Gothic ProN", "Hiragino Sans", Meiryo, sans-serif` のような標準的なゴシック体ベース。
- **装飾ルール**: **絵文字（Emoji）は使用しない**。アイコンが必要な場合は、CSSでの図形描写やシンプルなSVGを使用する。
- **UI / UX**:
  - **選択肢のUI**: 各選択肢の左側にクリック可能なエリア（×ボタン）を独立して配置。不要な選択肢をグレーアウト・取り消し線で除外できるようにし、誤タップを防ぐ。
  - **学習体験**: 選択肢（回答）をクリックすると、自動的に正解・解説が画面下部に展開し、スムーズにスクロールする（オートスクロール機能）。解説の冒頭に「正答は○番です」のような無駄な文章は省き、スッキリさせる。
  - **広告・リンクの配置**: 学習の妨げにならないよう、問題や解説の直下ではなく「ナビゲーション」のあとに配置する。

## 3. ディレクトリ・ファイル構成
サイトは静的サイトとしてビルドされ、GitHub Pagesで公開・運用されます。

```text
PEChemistrySite/
├── _dev/                   # 開発用ディレクトリ（ビルドスクリプト、ローカルドキュメント等）
│   ├── build_site.py       # （重要）サイトのHTMLを生成するPythonスクリプト
│   ├── SITE_GUIDELINES.md  # 本ドキュメント（ローカル専用、.gitignoreで除外）
│   └── __pycache__/
├── data/
│   ├── questions.json      # 過去問の全データ（問題文、選択肢、正解、解説、参考資料など）
│   └── changelog.json      # トップページ等に表示する更新履歴データ
├── .github/
│   └── workflows/
│       └── build.yml       # GitHub ActionsによるCI/CD設定（自動ビルド＆デプロイ）
├── .gitignore              # git管理から除外するファイルのリスト
├── styles.css              # サイト全体のデザイン定義
├── script.js               # ナビゲーションや問題ページのUIインタラクション制御
├── search.js               # クライアントサイドでの検索機能（questions.jsonをフェッチ）
├── logo.svg                # サイトのロゴやアイコン（SVG推奨）
└── index.html              # トップページ（ビルドスクリプトによって出力・上書きされる）
    ├── 2024/               # 年度ごとのディレクトリ（自動生成）
    │   ├── index.html      # 該当年度の問題一覧
    │   ├── q01/            # 個別の問題ページ（自動生成）
    │   │   └── index.html
    │   └── ...
```

## 4. 運用・保守手順
1. **コンテンツの追加**: 
   - 新しい年度や問題を追加する場合は、`data/questions.json` を編集する。
   - 変更履歴を残したい場合は `data/changelog.json` を追記する。
2. **ローカルでのビルドと確認**:
   - `python _dev/build_site.py` を実行してサイト全体を再構築する。
   - `python -m http.server 8080` を実行し、ブラウザ（`http://localhost:8080`）で表示崩れや動作を確認する。
3. **デプロイ**:
   - ローカルでの動作確認後、変更をコミットしてGitHubの `main` ブランチにプッシュする。
   - `.github/workflows/build.yml` により、GitHub Actionsが自動的に `build_site.py` を走らせ、最新の状態がGitHub Pagesにデプロイされる。
